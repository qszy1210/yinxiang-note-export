# 印象笔记导出工具 - 开发文档

## 笔记本/笔记选择逻辑重构

### 背景

原有逻辑存在以下问题：
1. 笔记选择状态没有按笔记本隔离，切换笔记本后勾选状态丢失
2. 切换笔记本时右侧全量刷新，体验不好
3. 勾选状态展示混乱，部分选中/全选/未选状态不清晰

---

## 第一期：笔记本选择逻辑重构 ✅ 已完成

> 📝 完整改动记录见 [CHANGES.md](./CHANGES.md)

### 目标
建立每个笔记本独立的笔记选择缓存，解决勾选状态丢失的问题。

### 改动内容

#### 1. 新增数据结构

```javascript
// StateManager 中存储
export.noteSelectionCache: {
  [notebookGuid]: [noteGuid1, noteGuid2, ...]
}
```

#### 2. 核心函数

| 函数 | 说明 |
|------|------|
| `getNoteSelectionCache()` | 获取缓存（确保是 Set 格式） |
| `saveNoteSelectionCache(cache)` | 保存缓存（转成可序列化数组） |
| `toggleNoteSelection(guid, checked)` | 勾选笔记时更新对应笔记本的缓存 |
| `updateNotebookCheckboxState(notebookGuid)` | 根据缓存更新左侧笔记本勾选状态 |
| `handleSelectAllNotes()` | 全选当前笔记本的笔记 |
| `updateSelectAllState()` | 更新全选框状态 |

#### 3. 交互逻辑

| 操作 | 效果 |
|------|------|
| 点击左侧笔记本 | 加载该笔记本笔记，从缓存恢复勾选状态 |
| 勾选笔记 | 更新该笔记本的缓存 + 刷新左侧勾选状态 |
| 顶栏全选 | 全选/取消全选当前笔记本的所有笔记 |
| 左侧显示 | 根据缓存自动显示：全选☑ / 部分选➖ / 未选☐ |

#### 4. 样式改动

- `.notebook-item.current` — 当前查看的笔记本（浅橙背景）
- `.notebook-item.partial` — 部分选中的笔记本（浅黄背景）
- `.checkbox-partial` — 部分选中时显示减号
- `.notebook-item .note-count` — 笔记本后显示选中数量

#### 5. 待完善

- [ ] 导出功能需要适配新的缓存结构（目前导出仍用旧的 `selectedNotes`）
- [ ] 左侧减号点击后的行为待实现（目前是空操作）

## 第二期：UI 刷新优化 ⏳ 待开发

### 目标
解决切换笔记本时右侧全量刷新、闪屏的问题。

### 方案设计

#### 1. 缓存所有笔记本的笔记

```javascript
// 新增缓存
export.notebookNotesCache: {
  [notebookGuid]: [note1, note2, ...]
}
```

#### 2. 首次访问时加载并缓存

- 点击某笔记本时，先检查缓存
- 缓存存在：直接切换显示
- 缓存不存在：加载笔记 → 存入缓存 → 切换显示

#### 3. DOM 复用，不全量替换

- 右侧笔记列表用 `display:none/ block` 切换，不用 `innerHTML` 全量替换
- 或者用虚拟列表，只渲染可见区域的笔记

#### 4. 预期效果

- 首次访问某笔记本：看到"加载中"（实际只加载一次）
- 再次访问：瞬时切换，无闪烁

## 未解决问题

### 1. 导出功能适配
第一期改动后，笔记选择通过 `noteSelectionCache` 按笔记本隔离存储，但 `handleExport()` 仍在使用旧的 `selectedNotes` 全集导出。需要：
- 修改导出逻辑，优先使用 `noteSelectionCache`
- 如果选中了整个笔记本，直接导笔记本
- 如果只选了部分笔记，逐个导出

### 2. 左侧部分选中(➖)点击行为
目前点击减号没有实际效果。建议：
- 点击减号 → 自动全选该笔记本
- 或弹出提示让用户选择"全选"或"取消全选"

### 3. 笔记本笔记数量显示
如果能获取到每个笔记本的笔记总数，左侧括号可以显示成 `已选中/总数`，如 `(3/10)`，更直观。
