# 印象笔记导出工具

## 项目概述

印象笔记 Web 导出工具，支持选择笔记本/笔记导出为 Markdown 或 HTML。

**技术栈：** 原生 JS + CSS，无框架

**关键文件：**
- `public/index.html` — 页面结构
- `public/js/app.js` — 入口初始化
- `public/js/modules/export.module.js` — 导出模块核心逻辑
- `public/js/api.js` — Evernote API 调用

## 改动规范（强制）

### ⚠️ 必须严格执行

#### 什么时候更新 Recent Changes

完成以下任一情况后**必须**更新：
- 新增功能
- 修改核心逻辑
- 修复重要 bug
- 界面交互改动

**不算改动的（不用更新）：**
- 拼写错误修正
- 注释整理
- 无关紧要的样式调整
- 仅影响自己的 debug 代码

#### Recent Changes 保留规则

- **保留最近 3 次改动**，超出后**覆盖最旧的那条**
- 或者**保留一周内**的记录，超出后**覆盖最旧的那条**
- 以先到为准（先超出3次就按次数，先超出一周就按时间）

#### 如何更新 Recent Changes

**在 `## Recent Changes` 区段顶部插入新记录：**

```markdown
### YYYY-MM-DD
- 改动1描述
- 改动2描述
```

**同时更新 docs/CHANGES.md（必须）：**

```markdown
## YYYY-MM-DD

### 改动标题
**改动内容：**
- 具体改动描述

**涉及文件：**
- file1
- file2
```

#### 流程示意

```
假设当前 Recent Changes 有 3 条：
  ### 2026-04-05  ← 最新
  ### 2026-04-02
  ### 2026-03-30  ← 最旧

4月8日完成新改动后，更新为：
  ### 2026-04-08  ← 新增，插入顶部
  ### 2026-04-05
  ### 2026-04-02  ← 覆盖了 2026-03-30

同时 docs/CHANGES.md 顶部插入完整的 2026-04-08 记录
```
